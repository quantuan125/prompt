# PLAN: T102 (CWD) — Phase 1 / Stream `T102-PH001-ST002`: T102-STD Baselining & Adoption Closure

## I. EXECUTIVE SUMMARY

**Purpose**: Baseline the initiative standards system by closing adoption/verification gaps across the post-amendment `T102-STD-001..009` set and aligning SSOT references so standards remain enforceable, discoverable, and resistant to drift.

**Why this stream exists**: Stream 1 defined the Phase 1 "where-spec-lives" model (Model D) and the STD-contains-CLAUSE governance model. Stream 5 (ST005) drafted research-driven clause amendments to STD-001, STD-003, STD-005, STD-006, and STD-007. This stream turns the post-amendment standards landscape into a consistent, baselined standards layer: adoption verification, verification expectations, and SSOT reference hygiene.

**Execution deviation acknowledgment**: ST002 was originally intended to run before ST005 (establishing the baseline before amendments). In practice, ST004 research drove ST005 scoping directly, causing an out-of-sequence execution. ST002 now operates on the **post-amendment** STD surface. AC000 explicitly accounts for this deviation by inventorying what ST005 already addressed.

**Phase 1 decision dependencies (locked)**:
- Model D (Combined STD+DR Files): STD bodies + embedded `Specification/CLAUSE` + nested `Decision Record` live in one combined file per STD under `prompt/artifacts/tasks/T102/consultant/standards/`.
- STD-contains-CLAUSE: the combined file **is** the STD; CLAUSEs use `STDCID` addressing (e.g., `T102-STD-004-CLAUSE-001`).
- Concept STD indexes link by repo-relative **Canonical Path** (whole-file granularity).
- ST002 execution is deferred until **ALL** ST005 GATE-001 approvals are recorded (DEC002 from `T102-PH001-SES001`).
- ST001-AC008 (STD-004 self-compliance audit) SHOULD complete before ST002 execution to ensure the exemplar pattern is hardened.

> **Cross-stream dependency**: ST002-AC000 depends on all ST005 activities reaching GATE-001 approval. ST002-AC004 (SSOT alignment) must also account for ST006 outputs (Concept hygiene + I&R aggregation).
