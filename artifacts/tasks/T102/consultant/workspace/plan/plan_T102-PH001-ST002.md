# PLAN: T102 (CWD) — Phase 1 / Stream `T102-PH001-ST002`: T102-STD Baselining & Adoption Closure

## I. EXECUTIVE SUMMARY

**Purpose**: Baseline the initiative standards system by closing adoption/verification gaps across `T102-STD-001..009` (as indexed in SPS) and aligning SSOT references so standards remain enforceable, discoverable, and resistant to drift.

**Why this stream exists**: Stream 1 defines the Phase 1 “where-spec-lives” model (Model D) and the Concept-owned ADR Index linking contract (canonical paths). This stream turns that design into a consistent standards layer: adoption pointers, verification expectations, and SSOT reference hygiene.

**Phase 1 decision dependency (locked by Stream 1)**:
- Model D (Combined ADR+Spec Files): ADR bodies + embedded `Specification/CLAUSE` live in one combined file per ADR under `prompt/artifacts/tasks/T102/consultant/standards/`.
- Concept ADR indexes link by repo-relative **Canonical Path** (whole-file granularity; anchors not required in Phase 1 indexing).

> **Blocking Note (AC006)**: This stream is blocked until `T102-PH001-ST001-AC006` (STD-Contains-CLAUSE Governance Migration) completes. AC006 migrates from `ADR-contains-CLAUSE` to `STD-contains-CLAUSE` addressing — the combined file IS the STD (containing both `## Specification` and `## Decision Record`). STD adoption semantics change accordingly: `STD` is the combined file itself, not a separate document that "adopts" an ADR. AC002 task scope must reflect this once unblocked.
