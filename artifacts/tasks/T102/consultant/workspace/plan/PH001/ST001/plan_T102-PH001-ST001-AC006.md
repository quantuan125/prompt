# PLAN: T102 (CWD) — Phase 1 / Stream ST001 / Activity `T102-PH001-ST001-AC006`: STD-Contains-CLAUSE Governance Migration

## I. EXECUTIVE SUMMARY

**Purpose**: Migrate the governance model from `ADR-contains-CLAUSE` to `STD-contains-CLAUSE` so normative specifications (CLAUSEs) derive authority from **Standards (STD)** rather than Decision Records (ADR). This activity rebuilds the golden exemplar as `T102-STD-004`, updates ID taxonomy/semantics to support `STDCID`, aligns derivative artifacts (guideline, template, SPS, Concept indexes), and produces a controlled migration procedure for downstream use in ST003.

**Non-goal**: This activity does not execute the broader ST003 rollout (migrating all remaining combined files) or baseline all `T102-STD-*` artifacts (handled by `T102-PH001-ST003` and `T102-PH001-ST002` respectively).

---

**Activity ID**: `T102-PH001-ST001-AC006`  
**Objective**: Implement the approved Option C migration (STD-contains-CLAUSE), including exemplar rebuild + taxonomy updates + index realignment + advisory + validation, with a documented procedure usable for ST003.  
**Execution Mode**: SEQUENTIAL

**Key decisions (approved 2026-02-06)**:

5. **ST003 migration scope**: All scope levels (initiative + epic) within T102 only
6. **Migration script/procedure**: Develop on `T102-STD-004` (this activity), reuse for T102-only in ST003, then all initiatives later
