# PLAN: T102 (CWD) — Phase 1 / Stream `T102-PH001-ST006`: Option (c) Transition Execution (Concept I&R Aggregation + Hygiene)

## I. EXECUTIVE SUMMARY

**Purpose**: Execute the **minimum operational transition** required after locking **Option (c)** for Issues & Risks (I&R) hosting:

- **Canonical I&R remains in SPS** (Initiative + Epic scope).
- **Concept hosts an aggregation dashboard** (non-normative; pointers-only) for cross-scope visibility.

This stream exists because the Phase 1 Standards Amendment stream (`T102-PH001-ST005`) drafts amended clauses, but does **not** perform SSOT rollout/hygiene work. ST006 is the bounded execution stream that makes Option (c) operational while keeping changes gated by the approved standards amendments.

**Scope**:
- In scope:
  - Repair known Concept register link-integrity drift (hygiene prerequisite).
  - Add a Concept I&R aggregation register **in Concept body** (pointers-only; non-normative).
  - Add minimal SPS pointer blocks to the Concept aggregation register (link-don’t-duplicate).
  - Issue a communication brief to the T102A owner describing Option (c) impacts and planning triggers.
- Out of scope:
  - Drafting/amending STD clauses (ST005 scope).
  - SPS modularization (deferred; trigger-based).
  - Any automation/tooling to sync registers (manual verification only per minimal-automation constraints).

**Execution mode**: GATED — ST006 activities that introduce/expand Concept-as-audit-surface MUST be treated as non-conformant until the relevant ST005 amended standards are approved.
