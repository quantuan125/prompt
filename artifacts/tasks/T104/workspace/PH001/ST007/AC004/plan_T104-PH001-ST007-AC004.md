**Deliverable**:
- Enhanced `validate_initiative_structure.py` with `--profile` argument

**Inputs Required**:
- `prompt/scripts/validate_initiative_structure.py` - current source

**Scope**:
- In scope:
  - `--profile pre-migration` flag
  - Suppress checks: `_validate_no_type_first_dirs`, `_validate_no_root_raw` when profile is `pre-migration`

    - Missing SSOT `concept_*.md` remains an error
- Out of scope:

  - Changes to existing check logic (only conditional gating)

**Steps**:
1. Add `--profile` CLI argument with choices `[post-migration, pre-migration]`, default `post-migration`
2. Pass profile to `InitiativeValidator.__init__`
3. Gate `_validate_no_type_first_dirs` and `_validate_no_root_raw` calls on profile != `pre-migration`
4. Verify: run against P with `--profile pre-migration` (expect reduced noise, but hard blockers still present until remediated); run against T104 with default profile (expect 0 errors - no regression)

**Success Criteria**:
