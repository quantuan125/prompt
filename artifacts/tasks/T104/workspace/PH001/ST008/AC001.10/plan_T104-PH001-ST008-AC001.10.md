# PLAN: T104 (CWS) - Phase 1 / Stream T104-PH001-ST008 / Activity `T104-PH001-ST008-AC001.10`: IMPLEMENTATION Spec Precision, External-Review Sufficiency, Role-Boundary Reconciliation & Pre-Implementation Artifact Governance

## I. EXECUTIVE SUMMARY

**Purpose**: Close the governance gaps exposed by the AC004 Gate-002 QA remediation so a developer-facing implementation specification cannot be approved as part of a gate package unless it is actually included, structurally specific, independently assessed for commissionability, tied to a single authoritative review surface, and clearly separated from premature downstream execution or proposal-level convention work.

**Trigger**: The AC004 QA path exposed eight failure modes that must be closed permanently:
- the implementation spec with execution audience `developer` was not truly part of the Gate-001 package before approval
- `guideline_workspace_implementation.md` allowed vague `Implementation Detail`
- the AC004 external review did not detect the missing/weak implementation spec and did not assess whether each SPEC was independently commissionable
- high-level workspace rules conflicted with IMPLEMENTATION-family guidance on whether the consultant may author `task_specification` artifacts
- the workspace suite did not explicitly state when a pre-implementation concrete artifact must be carried as a `standards_input` proposal instead of being materialized before gate approval
- a gate package could surface multiple review-like analyses without naming which one is authoritative for client disposition
- proposal/GDR language did not force an explicit split between consultant-only reminder surfaces and broader role-based operational protocols
- same-gate QA remediation work could be folded into live artifacts without an explicit plan/session/package tracking requirement

**Non-goal**: Do not mutate the AC004 successor package directly. AC001.10 hardens the governance surfaces that control how successor implementation packages are authored, reviewed, and dispositioned.
