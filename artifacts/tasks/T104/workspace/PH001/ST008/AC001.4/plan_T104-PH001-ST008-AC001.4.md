## I. EXECUTIVE SUMMARY

**Purpose**: Establish research-backed governance rules for classifying gate impacts (internal vs. external), determining the correct gate response (recycle vs. supersession vs. reopening), extending the GDR lifecycle to handle baseline change events, and providing retroactive application guidance for `P-PH000-ST002-AC002`.

### Task TK003: Produce Industry-Grounded Assessment Analysis for Gate Impact Classification

**Task ID**: `T104-PH001-ST008-AC001.4-TK003`

**Purpose**: Evaluate gate impact classification models from industry frameworks and propose a governance model for the workspace suite. This is a lightweight `analysis_type: 'assessment'` artifact, not a commissioned research package.

**Deliverable**:
- `prompt/artifacts/tasks/T104/workspace/PH001/ST008/AC001.4/analysis/analysis_T104-PH001-ST008-AC001.4_gate-impact-classification-assessment.md`

**Scope**:
- In scope:
  - Impact taxonomy: internal rework vs. external baseline change
  - Decision test: when does external impact warrant same-gate reassessment vs. gate supersession vs. gate reopening?
  - GDR lifecycle extension: what new event types or disposition values are needed?
  - Gate disposition types for externally-impacted gates (new Client Decision enum values if needed)
  - Analysis artifact deprecation model: how to mark superseded gate artifacts (frontmatter status, Evidence Index, Links Register)
  - Artifact status enum extension: whether `superseded` needs to be added to the canonical artifact status vocabulary
  - Edge case coverage (all 7 scenarios from consultation):
    1. Pending gate + external impact (the AC002 case)
    2. Approved gate + external impact (gate reopening)
    3. Gate in recycle + external impact (compound impact)
    4. External impact on inputs only (same decision boundary)
    5. External impact on decision boundary (different question)
    6. Cross-initiative external impact (standard amendment affecting multiple initiatives)
    7. Cascading external impact (one gate's resolution triggers impact on dependent gate)
  - Retroactive application assessment for `P-PH000-ST002-AC002`
- Out of scope:
  - Implementing guideline patches (that's TK005–TK008)
  - Restructuring the AC002 gate package (that's a downstream consumer)
  - Changes to gate structure rules unrelated to impact classification

**Inputs Required**:
- `prompt/templates/consultant/workspace/guideline_workspace_plan.md` — current §VI.K recycle rules, §VI.D gate status enum
- `prompt/templates/consultant/workspace/guideline_workspace_proposal.md` — current §VII GDR specification, Client Decision enum, GDR lifecycle
- `prompt/templates/consultant/workspace/guideline_workspace_verification.md` — current §VII Situation A/B rework paths
- `prompt/templates/consultant/workspace/workspace_documentation_rules.md` — current §7 workflow chain
- `prompt/artifacts/tasks/P/workspace/PH000/ST002/AC002/` — live exemplar package (plan, proposal, analysis artifacts, session notes)
- Industry frameworks: PRINCE2 exception management, Cooper Stage-Gate recycle/kill/re-scope, PMI PMBOK 7 change request classification, NASA NPR 7120.5 KDP re-baselining, ISO 21502 change management

**Success Criteria**:
